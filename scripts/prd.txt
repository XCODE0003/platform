# 📊 Торговая Платформа - MVP

## 🎯 Обзор
Торговая платформа для криптовалют, фиата, металлов и акций с минимальным набором функций для запуска.

**Текущее состояние:**
- ✅ Laravel + Vue.js + Inertia.js
- ✅ Filament admin
- ✅ Базовая система кошельков
- ✅ Westwallet интеграция

---

## 📋 Функционал

### 1. 🔐 Аутентификация

#### 1.1 Регистрация
- Email + пароль
- Email верификация
- Автоматическое создание кошельков

#### 1.2 Вход
- Email + пароль
- Опциональная 2FA проверка

#### 1.3 Восстановление пароля
- Отправка ссылки на email
- Форма установки нового пароля
- Обязательный logout со всех устройств

#### 1.4 Google 2FA (G2FA)
- Включение/отключение в настройках
- QR код для Google Authenticator
- Резервные коды (10 штук)
- Проверка кода при входе (если включен)
- Обязательна для вывода средств

#### 1.5 Смена пароля
- Форма: текущий пароль + новый пароль
- Email уведомление после смены
- Logout со всех сессий

### 2. 💰 Балансы и Транзакции

#### 2.1 Пополнение баланса

**Crypto (Westwallet) ✅:**
- Генерация адреса депозита
- QR-код адреса
- Предупреждение о сети (ERC-20/TRC-20)
- IPN callback зачисление
- Минимальная сумма

**Fiat (1 платежка):**
- Выбрать: Stripe / Unlimint / другая
- Депозит картой (Visa/Mastercard)
- 3D Secure
- Callback обработка
- Мин: $10, Макс: $10,000

#### 2.2 Подача заявки на вывод
- Форма: валюта, сумма, адрес/реквизиты
- 2FA обязательна
- Email подтверждение (ссылка)
- Статусы: pending, processing, completed, cancelled
- Отмена до обработки админом

#### 2.3 История транзакций
- Таблица всех операций
- Колонки: дата, тип (deposit/withdrawal/trade), валюта, сумма, статус, TX ID
- Фильтры: тип, валюта, статус, дата
- Пагинация

#### 2.4 Вывод адресов currency
- Список адресов депозита для всех крипто
- Отображение: валюта, адрес, сеть, QR-код
- Кнопка "Копировать адрес"

### 4. 📈 Торговля

#### 4.1 TradingView Терминал
- Встроенный TradingView widget
- Настройка через admin: включить/выключить элементы
  - Индикаторы (on/off)
  - Таймфреймы (какие доступны)
  - Типы графиков (свечи, линия и т.д.)
  - Рисование на графике (on/off)
  - Боковая панель (on/off)
- Данные из НАШЕГО вебсокета (не TradingView)

#### 4.2 Открытие сделок
- Форма покупки/продажи
- Поля:
  - Тип: Market (только это в MVP)
  - Количество (сумма отдаем)
  - Получим (расчет автоматически с учетом спреда)
  - Комиссия (показать)
- Кнопки: BUY / SELL
- Проверка баланса
- Подтверждение модалкой
- Исполнение мгновенно по нашей цене

#### 4.3 Модальное окно выбора пар
- Открывается при клике на текущую пару
- Список всех пар с группировкой
- Категории: Crypto, Fiat, Metals, Stocks
- Поиск по названию/символу
- Отображение: иконка, название, цена, изменение 24h
- Избранные (звездочка)

#### 4.4 Открытые и закрытые ордера
**Открытые ордера (таблица):**
- Пара, тип (buy/sell), цена открытия, количество, время
- Текущий P&L (прибыль/убыток)
- Кнопка "Закрыть"

**Закрытые ордера (история):**
- Пара, тип, цена открытия, цена закрытия, количество, P&L, время
- Фильтры: пара, тип, дата

#### 4.5 Последние транзакции (ФЕЙК)
- Виджет под графиком
- Список: время, цена, количество, сторона (buy/sell)
- Генерируется рандомно для "живости"
- Цвета: зеленый buy, красный sell
- Auto-scroll

#### 4.6 Order Book (ФЕЙК)
- Виджет: Asks (вверху), Bids (внизу)
- Колонки: цена, количество, сумма
- Цвета: красный asks, зеленый bids
- Генерируется рандомно вокруг текущей цены
- Обновление по вебсокету

### 5. 💹 Система Курсов

#### 5.1 Подключение своих курсов
- Admin может выбрать источник цены для каждой currency:
  - Binance API
  - Manual (ручной ввод)
  - Custom API (свой endpoint)
- Для Manual: форма ввода цены в USD
- Для Custom API: endpoint URL, метод парсинга JSON

#### 5.2 Сохранение курсов в БД
**Таблица `price_history`:**
- currency_id
- price (к USD)
- timestamp
- source (binance/manual/custom)

**Обновление:**
- Cron каждую минуту для автоматических
- Сохранение каждого обновления
- Хранить за последние 30 дней (OHLCV)

#### 5.3 Построение графика через наш вебсокет
**WebSocket сервер:**
- Отдает данные для TradingView
- OHLCV данные из нашей БД (price_history)
- Real-time обновления цены
- Управление спредом на лету

**Спред:**
- Admin настройка спреда по парам (%)
- Buy price = base_price * (1 + spread/100)
- Sell price = base_price * (1 - spread/100)
- Отображение spread в виджете

### 6. 💬 Чат и Поддержка

#### 6.1 Чат с тикетами (юзер сторона)
**Страница /support:**
- Список тикетов юзера
- Создание нового тикета
  - Тема
  - Категория (депозит/вывод/торговля/другое)
  - Описание проблемы
  - Прикрепление файлов (скриншоты)
- Статусы: open, replied, resolved, closed

**Страница тикета /support/{id}:**
- Чат интерфейс
- История сообщений
- Поле ввода + отправка
- Real-time обновления (Pusher/WebSocket)
- Индикатор "админ печатает..."
- Прикрепление файлов

#### 6.2 Чат в админ панели (Filament)
**Отдельное приложение чата:**
- Независимое Laravel приложение или SPA (Vue)
- Endpoint: /admin-chat
- Авторизация по токену админа

**Интеграция в Filament:**
- Custom Page: `app/Filament/Pages/SupportChat.php`
- Iframe на /admin-chat с передачей auth токена
- Full screen режим
- Список всех тикетов
- Фильтры: статус, категория
- Счетчик новых сообщений
- Выбор тикета → чат справа
- Ответ на сообщения
- Закрытие тикета
- Внутренние заметки (не видны юзеру)
